# Vuex

Vuex 是一个专为 Vue 应用程序开发的状态管理模式，它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

通过如下命令安装 Vuex

```bash
cnpm install vuex --save
```

## 1 入门

通过一个计数器小案例，简单认识一下 Vuex。

**步骤一：** 在项目根目录下新建文件夹 vuex，在 vuex 文件夹中新建文件 store.js，用于管理状态

```javascript
import Vue from 'vue'
import Vuex from 'vuex'
Vue.use(Vuex)

// state是状态对象
const state = {
  count: 250
}

// 修改状态数据的方法
const mutations = {
  // 传递参数，第一个参数是固定的state对象，后面可以传递自定义的参数
  add(state, n) {
    state.count += n
  },
  reduce(state) {
    state.count--
  }
}

export default new Vuex.Store({ state, mutations })
```

**步骤二：** 在模板页面中显示状态、修改状态

```vue
<template>
  <div>
    <h2>{{ msg }}</h2>
    <p>
      <!-- 第一个参数是方法名，后面可以传入自定义的参数 -->
      <button @click="$store.commit('add', 10)">+</button>
      <button @click="$store.commit('reduce')">-</button>
    </p>
    <div>{{ $store.state.count }}</div>
  </div>
</template>
<script>
  import store from '@/vuex/store'
  export default {
    data() {
      return {
        msg: 'Hello Vuex'
      }
    },
    store
  }
</script>
```

**补充：** 单文件组件（上面所说的模板页面）和全局的 component 方法定义组件，data 属性是一个函数，并且该函数返回了一个对象，为什么不能直接把对象赋值给 data 呢？

> 当一个组件被定义，data 必须声明为返回一个初始数据对象的函数，因为组件可能被用来创建多个实例。如果 data 仍然是一个纯粹的对象，则所有的实例将共享引用同一个数据对象！通过提供 data 函数，每次创建一个新实例后，我们能够调用 data 函数，从而返回初始数据的一个全新副本数据对象。

使用一个函数来返回一个对象可以保证每个组件中的 data 都是独立的。

## 2 读取状态

上一节的计数器小案例中，我们通过 `<div>{{ $store.state.count }}</div>` 来读取并显示状态，这样并不好，我们想把 `$store.state.count` 换成 `count`，下面有三种方案：

```vue
<template>
  <div>{{ count }}</div>
</template>
<script>
  import store from '@/vuex/store'
  import { mapState } from 'vuex'
  export default {
    // 方案1
    // computed: {
    //   count() {
    //     return this.$store.state.count
    //   }
    // },

    // 方案2
    // computed: mapState({
    //   count: state => state.count
    // }),

    // 方案3，最常用
    computed: mapState(['count']),
    store
  }
</script>
```

## 3 改变状态

第一节中，我们是通过 `<button @click="$store.commit('add', 10)">+</button>` 来改变状态的，但是这样写不太好，我们想把 `$store.commit('add', 10)` 换成 `add(10)`，需要引入 mapMutations。

```vue
<template>
  <div>
    <p>
      <button @click="add(10)">+</button>
      <button @click="reduce">-</button>
    </p>
    <div>{{ count }}</div>
  </div>
</template>
<script>
import store from '@/vuex/store'
import { mapState, mapMutations } from 'vuex'
export default {
  computed: mapState(['count']),
  methods: mapMutations(['add', 'reduce']),
  store
}
</script>
```

## 4 getters

getters 是状态管理中的计算过滤操作，有点像 Vue 构造器中的 computed 选项，每次状态发生改变，都会走一下 getters。

**步骤一：** store.js 中，加上 getters

```javascript
import Vue from 'vue'
import Vuex from 'vuex'
Vue.use(Vuex)

const state = {
  count: 250
}

const mutations = {
  add(state, n) {
    state.count += n
  },
  reduce(state) {
    state.count--
  }
}

const getters = {
  count: state => (state.count += 500)
}

export default new Vuex.Store({ state, mutations, getters })
```

**步骤二：** 模板页面中

```vue
<template>
  <div>
    <p>
      <button @click="add(10)">+</button>
      <button @click="reduce">-</button>
    </p>
    <div>{{ count }}</div>
  </div>
</template>
<script>
  import store from '@/vuex/store'
  import { mapState, mapMutations, mapGetters } from 'vuex'
  export default {
    computed: {
      // 注意扩展运算的使用
      ...mapState(['count']),

      // count() {
      //   return this.$store.getters.count
      // }
      // 简写
      ...mapGetters(['count'])
    },
    methods: mapMutations(['add', 'reduce']),
    store
  }
</script>
```

## 5 actions

actions 用于异步修改状态。上面的 mutations 是用于同步修改状态。

**步骤一：** store.js 中，加上 actions

```javascript
import Vue from 'vue'
import Vuex from 'vuex'
Vue.use(Vuex)

const state = {
  count: 250
}

const actions = {
  addAction(context) {
    context.commit('add', 100)
    setTimeout(() => {
      context.commit('reduce')
    }, 5000)
    console.log('看看reduce什么时候执行')
  },
  reduceAction({ commit }) {
    commit('reduce')
  }
}

export default new Vuex.Store({ state, actions })
```

**步骤二：** 模板页面中

```vue
<template>
  <div>
    <p>
      <button @click="addAction">+</button>
      <button @click="reduceAction">-</button>
    </p>
    <div>{{ count }}</div>
  </div>
</template>
<script>
import store from '@/vuex/store'
import {
  mapState,
  mapActions
} from 'vuex'
export default {
  computed: {
    ...mapState(['count'])
  },
  methods: {
    ...mapActions(['addAction', 'reduceAction'])
  },
  store
}
</script>
```

## 6 模块组

模块组其实就是把 state、mutation、getters、actions 放到一起。

**步骤一：** store.js 中

```javascript
// ...
const moduleA = { state, mutations, getters, actions }

export default new Vuex.Store({ modules: { moduleA } })
```

**步骤二：** 模板页面中只需要修改一下读取状态的方式

```vue
<template>
  <div>
    <!-- 如果下面的computed没有设置count的话，就用下面这种插值方式 -->
    <!-- <div>{{ this.$store.state.moduleA.count }}</div> -->
    <div>{{ count }}</div>
  </div>
</template>
<script>
  import store from '@/vuex/store'
  export default {
    computed: {
      count() {
        return this.$store.state.moduleA.count
      }
    },
    store
  }
</script>
```
